# AGENTS.md

## LLM Wiki Capture

- Knowledge base root: this repository root.
- Clone URL: configure your hosted Git URL if this repository should be cloned
  automatically on another machine.
- Integration branch: main.
- Read first: purpose.md, schema.md, wiki/index.md, wiki/log.md.
- Update after knowledge edits: wiki/index.md, wiki/log.md.
- Require clean worktree before capture: yes.
- Commit policy: no-commit until you are comfortable with the workflow; then
  choose commit-only or commit-and-push.
- Commit identity: configure a real name and email before enabling commits.
- If raw session evidence or user-provided source material is unavailable,
  downgrade to review and do not edit this knowledge base.

## Capture Policy

- Save reusable decisions, constraints, pitfalls, commands, domain facts, and
  operating rules.
- Do not save secrets, tokens, credentials, noisy logs, temporary progress, or
  unsupported guesses.
- Prefer updating an existing owner page over creating a new page.
- Create a new page only when the knowledge would be hard to find inside an
  existing owner.
- Keep provenance clear enough that a later reader can tell where each claim came
  from.
